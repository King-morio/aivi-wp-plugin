User Guide

The cleanest documentation surface for AiVI should feel less like a support dump and more like a guided operating system for the plugin. The user guide becomes the default landing article because it answers the most practical first question: how do I use this well inside WordPress?

From here, a reader can move naturally into the check reference, troubleshooting, privacy, or contributor docs without leaving the settings environment. That keeps the experience self-serve, premium, and trustworthy.

Analyze contentStart from Gutenberg or Classic Editor and understand what AiVI is scoring.
Read the railLearn what surfaced findings mean, and what stays quiet unless needed.
Rerun safelyUnderstand stale results, superseded runs, and overlay draft behavior.

This surface also makes the support-side “Documentation” button more useful. Instead of sending people to a generic external page, it can deep-link directly into the most relevant article:

Billing & Plans → Pricing/help docs
Connection & Setup → Connection section
Analysis & Results → Troubleshooting or Check Reference
General Support → User Guide / Support guide
