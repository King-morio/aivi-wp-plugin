# AiVI Settings Option C Polish Track

## Why this exists

We approved **Option C / WP-Native Plus** as the right direction for the next settings-page polish pass.

This work is intentionally narrow:

- keep **Plans spotlight** untouched
- keep **Support center** untouched
- improve only the weaker surfaces:
  - shared settings-shell consistency
  - **Credit packs** tab
  - **Connection** tab callout under **Add another site to this account**

The goal is to make the settings page feel more cohesive and more mature without pushing it away from WordPress-native expectations.

## Approved direction

- Use the existing gold/golden hue system as a **disciplined accent language**
- Do **not** spread strong gold fills everywhere
- Make **Credit packs** cleaner and more product-like
- Make **Connection** guidance read like a proper checklist
- Avoid touching the approved **Plans spotlight** and **Support center** modules

## Milestones

### M1 — Shared shell consistency
- Lightly unify the settings shell using the approved accent family
- Improve tabs, section eyebrows, cards, and callouts where they currently feel plain or disconnected
- Keep the treatment subtle and WordPress-friendly

**Status:** Complete

- Added a restrained gold accent treatment to the shared settings shell without altering the approved Plans spotlight or Support center modules
- Refined tabs, meta chips, cards, callouts, and offer tags so the weaker tabs now feel more visually aligned with the premium surfaces
- Kept the treatment lightweight enough to remain comfortable inside wp-admin

### M2 — Credit packs cleanup
- Replace the raw-looking credit pack presentation with cleaner product-card treatment
- Improve offer hierarchy, spacing, and badges
- Keep the content and actions intact while making the surface feel more mature

**Status:** Complete

- Reframed credit packs as cleaner product cards with better hierarchy, subtle category kickers, and a stronger default-pack signal
- Preserved the existing actions and billing behavior while improving readability and perceived quality
- Kept the presentation calmer than a full sales layout so it still feels native to the settings page

### M3 — Connection callout clarity
- Rework the **Add another site to this account** callout so the action path is easier to scan
- Use structured list/checklist presentation instead of loose prose + bullets
- Keep the underlying flow and token behavior intact

**Status:** Complete

- Converted the callout into a clearer two-column layout with a step-by-step checklist and a dedicated token card
- Preserved the existing token reveal/copy behavior and support request path
- Left the underlying multi-site connection flow unchanged while making it much easier to follow

### M4 — Validation and packaging
- Run focused PHP and diagnostics validation
- Package a fresh plugin zip for specimen-site review
